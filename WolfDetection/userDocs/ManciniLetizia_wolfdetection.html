LABORATORIO DI INGEGNERIA DEI SISTEMI SOFTWARE

Introduction

Si vuole esplorare come sia possibile costruire modelli eseguibili di un sistema software partendo dall'analisi del problema, senza entrare subito nei dettagli tecnologici necessari per la costruzione effettiva del software.

Il caso di studio è una applicazione IOT denominata WolfDetection.

Requirements

Requisiti dati dal committente

Requirement analysis

Wolf Detection il servizio che si richiede di progettare
Led il sistema comprende due Led, uno rosso e uno blu
DMIN distanza minima di rilevamento di un ostacolo
Sonar HC-SR04 dispositivo connesso ad un Raspberry Pi
  • tramite comandi impartiti dalla stazione, si deve poter accendere e spegnere in remoto
  • quando rileva un ostacolo a distanza minore di DMIN, accende il Led blu
Fotocamera scatta una foto dell'ostacolo rilevato dal sonar e la invia ad un elaboratore per stabilire se l'ostacolo sia o meno un lupo

Use cases and scenarios

  1. Quando il Sonar rileva un ostacolo a distanza inferiore a un limite prefissato DMIN, il servizio accende il Led blu
  2. La fotocamera scatta una fotografia dell’ostacolo rilevato a distanza inferiore a DMIN e la invia ad un elaboratore di immagini in grado di capire se l’ostacolo è un lupo
  3. Se è stato rilevato un lupo, si accende il Led rosso
  4. Il Led blu e il Led rosso si spengono quando il sonar non rileva più ostacoli a distanza inferiore a DMIN

Il committente fornisce

  • il metamodello QActor per la modellazione del sistema (si veda QakActors24 per maggiori informazioni)

Problem analysis

Per realizzare un primo modello del sistema sulla base delle analisi, si sceglie di utilizzare il linguaggio di modellazione Qak fornito dal committente. Il metamodello

  • consente di catturare gli aspetti essenziali del sistema
  • offre l'astrazione QActor per rappresentare le entità come componenti autonomi ed indipendenti.
La Software Factory definita per il linguaggio crea automaticamente un modello eseguibile in Kotlin.
